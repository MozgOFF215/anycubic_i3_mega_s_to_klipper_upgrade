# Перенос системы Orange Pi с microSD на eMMC (Armbian)

Эта инструкция описывает безопасный перенос уже настроенной системы Orange Pi с `microSD` во внутреннюю память `eMMC`.

## Когда переносить

Перенос лучше делать только после того, как Klipper/Moonraker/Mainsail и сеть стабильно работают на системе с SD-карты.

## Что нужно заранее

- Текущая система загружается с `microSD`.
- `eMMC` определяется в системе.
- Занятое место в `/` помещается в объем `eMMC` (с запасом).
- Есть локальный доступ (монитор/клавиатура), на случай если после перезагрузки изменится IP.

## 1) Проверка источника и целевого диска

```bash
findmnt -no SOURCE /
lsblk -o NAME,SIZE,TYPE,MOUNTPOINT
```

Типичное состояние перед переносом:
- корень (`/`) на `mmcblk0p1` (microSD),
- внутренний eMMC виден как `mmcblk2` (обычно около 7-8G на Orange Pi 3 LTS с eMMC 8GB).

## 2) Проверка, что система влезает в eMMC

```bash
df -h /
sudo du -xhd1 /home
sudo du -xhd1 /var
```

При необходимости освободите место:

```bash
sudo apt clean
sudo journalctl --vacuum-time=7d
```

## 3) Запуск переноса через Armbian

```bash
sudo armbian-install
```

В меню выберите:
1. `Boot from eMMC - system on eMMC`
2. Целевой диск: `eMMC (/dev/mmcblk2)`
3. Файловая система: `ext4`

Подтвердите форматирование/запись на eMMC и дождитесь завершения.

## 4) Выключение и запуск уже с eMMC

```bash
sudo poweroff
```

После выключения:
- вытащите microSD,
- снова включите Orange Pi.

## 5) Проверка успешного переноса

```bash
findmnt -no SOURCE /
lsblk -o NAME,SIZE,TYPE,MOUNTPOINT
```

Критерии успеха:
- корень (`/`) находится на `mmcblk2p...` (eMMC),
- размер `/` соответствует размеру eMMC (для eMMC 8GB обычно около 7.0G доступного объема).

## Если без SD не загружается

1. Вставьте SD обратно и загрузитесь как раньше.
2. Снова выполните `sudo armbian-install`.
3. Выберите `Install/Update the bootloader on eMMC (/dev/mmcblk2)` (обновление только загрузчика).
4. Выключите, вытащите SD и проверьте запуск снова.

Дополнительно проверьте IP-адрес устройства после перезагрузки: DHCP может выдать новый адрес.
